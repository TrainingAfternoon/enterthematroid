Roadie

One of the biggest issues with reading web novels (novels published free lance on the internet, usually for free) is that they can be suddenly taken down if the author receives a corporate publishing deal. This can happen at any time, without warning. As a reader, I wanted to preserve some of my favorite stories for rereading… So I wrote a program for that.

Infrastructure

Roadie was my introduction into the world of networking and web infrastructure. As a very targeted web scraper, Roadie needed to be able to make requests to remote sites and process the response. It also needed to be able to parse (hopefully quickly) the HTML response for select elements, and extract them. I used the Beautiful Soup Python library to accomplish both of these tasks. Beautiful Soup provides a nice API for requesting HTML data from a static URL, and then parsing the DOM generated from that HTML file. From this project, I ended up learning way more about the DOM than I ever thought I would know… but now I’m certified web dangerous.

Roadie was one of the first real programming projects that I did on my own, without some kind of academic or professional imperative. It wasn't even intended to be a project at first-- it was supposed to be a quick script to get the job done. But as much as it suffers from that, the narrow constraints the code was written for give it a simplicity that makes it very easy to pick up and extend.

Looking at the work I've done for other projects, Roadie could work very well embedded in a whiptail cl dilaogue, or javaFX or WPF GUI.
