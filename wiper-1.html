Wiper

There are certain regulations for businesses, especially organizations that deal with sensitive information, for how they may dispose of hard drives securely. The most important is that the hard drive is wiped in a standards compliant way to ensure that the data is irretrievable. As an iCadet Intern with the School District of Milton, I researched different erasure protocols & tools, and designed a cost-effective automated process for wiping hard drives in compliance with the DoD-5220.22-m standard. The project was carried out solo over the course of one month.

Where We're Going We Don't Need SATA Adapters

One of the requirements for this project was that it could not cost money. So, we could not purchase sata adapters to interface with the loose collection of hard drives that we had. I circumvented this restriction by repurposing a lot of soon-to-be-recycled Dell computers, which had an onboard SATA capable for their optical disk drive. By removing the optical disk drive, the towers themselves became giant SATA adapters.

When in Doubt, Linux

One of the biggest issues the wiper project faced was that gnu scrub only ran on *nix environments due to how it treated file addresses. I learned how to make bootable linux drives to enable us to deploy to scrub the dell adapter towers.

Built on the back of GNU

To ensure standards-compliant erasure, Wiper is built on top of the scrub utility provided by the open source software titan, GNU (GNU is Not Unix!). Scrub handles all of the functionality of writing to the disk, and verifying the overwrites.

Whiptail Brings GUI to the Terminal

As a command line utility, GNU scrub can seem very daunting to use at a glance. Using the whiptail dialog library, which is built on top of newt, wiper actually implements a graphical interface right out of the 90s in the command line for ease of use. Wipe hard drives with naught but a few button presses!

Looking Back, and Looking Forward:
Analysis & Reflection

Wiper was ultimately successful, but it has some UI bugs which need to be resolved.

Improvements
  • Only searches /dev/ for drives
  • Some CL features not supported in GUI
  • Does not protect against wiping active drive
Successes
  • Graphical UI
  • Multiple Supported Standards
  • Semi-Smart Drive Detection
  • Learned Bash

Wiper ended up coming together nicely and it did exactly what I needed it to. Provide an easy way to wipe drives, with a interface that was friendly to those familiar with technology, but not necessarily the terminal, so that I could pass it off to my successors.

I learned the Bash shell language during my time working on this project, which has been an invaluable skill ever since. Bash is a commonly supported shell across all unix derivatives, including mac OSX until very recently. It allows working with compiled binaries and also comes with its own features for manipulating unix filesystems.

I also learned how to use the whiptail dialog library, which is the Debian replacement for the classic dialog which has the same purpose, built on top of newt, a very simple GUI library for small memory applications.

However, the GUI is not fully finished and does not support all of the features that it's command line cousin does. For example, it does not support dry runs. Wiper also only supports searching the dev directory for drives. Wiper also contains no safeties, which makes it very easy to very quickly mess up and delete your own c drive.
